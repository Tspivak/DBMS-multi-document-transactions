About

In this course of distributed database systems we chose the multi-document transaction process. The server side in Spring.io with mongoDB (Primary & Replica)...So we chose to perform the demo - On a transaction of transferring money from a client's account. Steps:
- Establishment of two sites, MongoDB, Primary and Replica.
- Creating a documentation update mechanism (CRUD) in various collections
- Developing a user interface (WEB) to display the process
